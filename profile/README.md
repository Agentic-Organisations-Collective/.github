# 🤖 Agentic Organisations Collective

👋 Welcome! Whether you're just curious, a member, or on the board — this is the
best place to start. It explains how we work and points you to all of our
repositories.

We're a collective exploring how organisations can be **designed, governed, and
run with agentic, AI-native workflows** — transparent by default and versioned
from end to end.

Our work lives in focused repositories, each with its own clear boundary. That
way legal records, member data, editorial planning, and public communication
never get mixed up. 🧩

## 🗂️ How we're organised

We're a GitHub organization made up of a few focused monorepos. We split them by
**access level first, topic second**:

- 🎯 **One job per repo** — each repository owns a single area and its own
  permissions.
- 📜 **Versioned & auditable** — legal texts go through review and become
  tagged, immutable releases.
- 🌍 **Public by default, private when needed** — sensitive member and board
  data stay safely separated.

We use four access levels:

| 🔐 Level | 👀 Who can read | 📦 Used for |
| --- | --- | --- |
| 🟢 **Public** | Everyone | Statutes, public communication, released documents |
| 🟡 **Member** | Members | Internal working content and documentation |
| 🔴 **Board** | Board | Member data, finances, formal administration |
| ⚫ **Secret / Vault** | External only | Credentials and tokens — never stored in a repo |

👉 Want the full picture of what goes where and why? Read the
[repository architecture charter](https://github.com/Agentic-Organisations-Collective/.github/blob/main/docs/repository-architecture.md).
